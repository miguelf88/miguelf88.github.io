For a transit agency, a market area represents the geographic location from which passengers could access their service. This project required me to delineate the market areas for PART. A Park & Ride survey completed in mid-2018 helped me accomplish this task. Using a free geocoder from Texas A&M University, I plotted the home location of PART's passengers who completed the survey. I then calculated the Euclidean distance from each home to the corresponding park & ride lot. Using the interquartile range method, I removed outliers from the dataset.

We wanted to analyze the demographics of the market areas. For this, I located tables from the 2013-2017 5-year estimates of the American Community Survey using American Fact Finder. Once the appropriate tables were found, they were joined to a block group level geography from TIGER/Line shapefiles using the GEOID as the field to join on. I created a new field in the dataset which I called "census_area". Using the field calculator I populated the new field with values from the "Shape_Area" field. This would allow me to calculate new values for the block groups once I performed an intersect.

Using the market areas feature class and the block group Census data as inputs, I performed an intersect. The output from this tool is a new feature class containing the Census block groups that are within the boundaries of the PART market areas. I added new fields to this feature class where new values for population, number of households, minority population, and population in poverty, as well as other demographics would be calculated.
